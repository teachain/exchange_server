## 1. 服务详细规格

### 1.1 MatchEngine (撮合引擎)

**服务端口**: 7316  
**数据库**: trade_log

#### 1.1.1 核心功能

- 订单匹配 (限价单、市价单、止损单、止损限价单)
- 订单簿管理 (价格优先、时间优先)
- 用户余额管理 (可用/冻结)
- Kafka事件发布

#### 1.1.2 HTTP端点

| 端点 | 方法 | 说明 |
|------|------|------|
| `/health` | GET | 健康检查 |
| `/order/create` | POST | 创建订单 |
| `/order/cancel` | POST | 取消订单 |
| `/order/:id` | GET | 获取订单 |
| `/balance/:user_id/:asset` | GET | 获取余额 |
| `/depth/:market` | GET | 获取深度 |
| `/asset/list` | GET | 资产列表 |
| `/asset/summary` | GET | 资产摘要 |

#### 1.1.3 订单类型

```go
type OrderType uint8
const (
    OrderTypeLimit     = 1  // 限价单
    OrderTypeMarket    = 2  // 市价单
    OrderTypeStopLoss  = 3  // 止损单
    OrderTypeStopLimit = 4  // 止损限价单
)

type OrderSide uint8
const (
    OrderSideAsk = 1  // 卖
    OrderSideBid = 2  // 买
)

type OrderStatus uint8
const (
    OrderStatusPending  = 1  // 待处理
    OrderStatusPartial  = 2  // 部分成交
    OrderStatusFinished = 3  // 已完成
    OrderStatusCanceled = 4 // 已取消
)
```

#### 1.1.4 费用结构

- 吃单方 (Taker): 0.1%
- 挂单方 (Maker): 0.1%

### 1.4 AccessHTTP (HTTP API网关)

**服务端口**: 9066  
**监控端口**: 8081

#### 1.4.1 JSON-RPC API

**请求格式**:
```json
{
  "method": "order.put_limit",
  "params": [1, "BTC_USDT", 2, "0.1", "50000", "0.001", "0.002", ""],
  "id": 1
}
```

**响应格式**:
```json
{
  "error": null,
  "result": {...},
  "id": 1
}
```

#### 1.4.2 API方法详细

##### 订单 API

**order.put_limit** - 下限价单
```
参数: user_id, market, side, amount, price, taker_fee, maker_fee, source
返回: order_id, trades
```

**order.put_market** - 下市价单
```
参数: user_id, market, side, amount, taker_fee, source
返回: order_id, trades
```

**order.cancel** - 取消订单
```
参数: user_id, market, order_id
返回: order_info
```

**order.book** - 获取订单簿
```
参数: market, side
返回: orders数组
```

**order.depth** - 获取深度
```
参数: market, limit
返回: bid/ask深度列表
```

**order.pending** - 获取待处理订单
```
参数: user_id, market, side, offset, limit
返回: orders数组
```

**order.deals** - 获取订单成交
```
参数: order_id, offset, limit
返回: deals数组
```

**order.finished** - 获取已完成订单
```
参数: user_id, market, side, start_time, end_time, offset, limit
返回: orders数组
```

**order.finished_detail** - 获取订单详情
```
参数: order_id
返回: order_info
```

##### 余额 API

**balance.query** - 查询余额
```
参数: user_id, [assets...]
返回: 余额映射
```

**balance.update** - 更新余额
```
参数: user_id, asset, business, change, detail
返回: 结果
```

**balance.history** - 余额历史
```
参数: user_id, asset, business, start_time, end_time, offset, limit
返回: 历史记录数组
```

##### 市场 API

**market.list** - 市场列表
```
参数: 无
返回: 市场数组
```

**market.summary** - 市场摘要
```
参数: market
返回: open, close, high, low, volume, deal
```

**market.last** - 最新价格
```
参数: market
返回: last_price
```

**market.deals** - 最近成交
```
参数: market, limit, last_id
返回: deals数组
```

**market.kline** - K线数据
```
参数: market, interval, start_time, end_time
返回: klines数组
```

**market.status** - 市场状态
```
参数: market, period
返回: open, close, high, low, volume, deal, last
```

**market.status_today** - 今日状态
```
参数: market
返回: 今日市场状态
```

**market.user_deals** - 用户成交
```
参数: user_id, market, offset, limit
返回: deals数组
```

##### 资产 API

**asset.list** - 资产列表
```
参数: 无
返回: 资产数组
```

**asset.summary** - 资产摘要
```
参数: 无
返回: 资产汇总
```

## 2. 数据库详细设计

### 2.1 trade_log 数据库

#### 2.1.1 operlog_YYYYMMDD 表

**用途**: 操作日志，按日期分区

```sql
CREATE TABLE operlog_YYYYMMDD (
    id      BIGINT UNSIGNED NOT NULL PRIMARY KEY,
    type    INT NOT NULL,
    time    DOUBLE NOT NULL,
    data    TEXT,
    INDEX idx_time (time)
);
```

**type枚举**:
- 1: OrderCreate (订单创建)
- 2: OrderDeal (订单成交)
- 3: OrderCancel (订单取消)
- 4: BalanceChange (余额变更)

#### 2.1.2 slice_balance 表

**用途**: 余额快照

```sql
CREATE TABLE slice_balance (
    id          INT UNSIGNED NOT NULL PRIMARY KEY AUTO_INCREMENT,
    slice_id    INT UNSIGNED NOT NULL,
    balance_data TEXT NOT NULL,
    FOREIGN KEY (slice_id) REFERENCES slice_history(id) ON DELETE CASCADE
);
```

#### 2.1.3 slice_order 表

**用途**: 订单快照

```sql
CREATE TABLE slice_order (
    id          BIGINT NOT NULL PRIMARY KEY AUTO_INCREMENT,
    slice_id    INT UNSIGNED NOT NULL,
    order_data  TEXT NOT NULL,
    FOREIGN KEY (slice_id) REFERENCES slice_history(id) ON DELETE CASCADE
);
```

#### 2.1.4 slice_history 表

**用途**: 分片进度记录

```sql
CREATE TABLE slice_history (
    id            INT UNSIGNED NOT NULL PRIMARY KEY AUTO_INCREMENT,
    timestamp     DATETIME NOT NULL,
    path          VARCHAR(512) NOT NULL,
    end_oper_id   BIGINT NOT NULL DEFAULT 0,
    end_order_id  BIGINT NOT NULL DEFAULT 0,
    end_deals_id  BIGINT NOT NULL DEFAULT 0,
    created_at    DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

```
+----------------+----------------+----------------+----------------+
| CRC32 (4B)    | Sequence (4B)  | ReqID (8B)     | BodySize (4B)  |
+----------------+----------------+----------------+----------------+
| ExtSize (2B)  |                                              |
+----------------+                                              |
| Body (Variable)|                                             |
+----------------+                                              |
| Ext (Variable) |                                             |
+----------------+                                              +
```
